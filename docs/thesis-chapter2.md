# 第二章 文獻探討

本章將回顧與本研究相關的國內外文獻，包括情緒分析理論、臉部表情辨識技術、語音情緒分析、文字情感分析以及多模態情緒識別等相關領域的最新進展。同時也會分析現有的類似產品，並說明本研究與現有研究之差異。

## 2.1 情緒分析相關研究

### 2.1.1 情緒的定義與分類

情緒（Emotion）是一種複雜的心理和生理狀態，涉及主觀體驗、生理反應以及行為表達等多個層面。自古以來，情緒一直是哲學、心理學以及神經科學等領域的重要研究課題。在學術研究上，不同的學者對於情緒有著不同的定義與分類方式。

Ekman 和 Friesen（1971）提出了六種基本情緒（Basic Emotions）的概念，包括快樂（Happy）、悲傷（Sad）、恐懼（Fearful）、驚訝（Surprised）、厭惡（Disgusted）以及憤怒（Angry）。這些基本情緒被認為是人類所共有的，且能夠透過臉部表情清晰地辨識。Ekman 的研究還開發了臉部動作編碼系統（Facial Action Coding System, FACS），該系統透過量化臉部肌肉的動作來描述臉部表情，為後來的臉部表情自動化辨識奠定了基礎。

除了基本情緒理論之外，Russell（1980）提出的情緒環狀模型（Circumplex Model）也對情緒研究產生了深遠的影響。該模型將情緒視為一個二維空間中的點，其中一個維度代表情緒的效價（Valence），即情緒的正負面向（如快樂 vs. 悲傷），另一個維度則代表情緒的覺醒程度（Arousal），即情緒的強烈程度（如激動 vs. 平靜）。這種連續性的情緒表示方式為後來的自動化情緒分析提供了一個有效的框架。

近年來，隨著深度學習技術的發展，情緒分析的研究重點逐漸從傳統的心理學理論轉向以資料驅動的機器學習方法。研究者們開始嘗試利用大量的標註數據來訓練能夠自動識別情緒的模型，這些模型往往能夠達到甚至超越人類的辨識準確率。

### 2.1.2 情緒構成理論

情緒構成（Emotion Construction）理論是近年來情緒研究領域的一個重要觀點。Barrett（2006）提出的建構主義觀點認為，情緒並非預先存在的心理類別，而是人類透過日常生活中對於身體感覺、行為反應以及環境脈絡的持續建構而形成的。這種觀點對於情緒分析系統的設計具有重要的啟示意義：它提示我們，情緒的識別不應該依賴於單一的線索，而應該綜合考量多種來源的資訊。

本研究正是基於情緒構成理論的觀點，採用多模態（Multimodal）的方法來分析使用者的情緒狀態。我們認為，透過結合臉部表情、語音特徵以及文字內容這三種不同類型的輸入，我們能夠獲得更加完整和準確的情緒評估結果。這種方法與傳統的單一模態分析方法相比，能夠更好地捕捉情緒的複雜性和多樣性。

### 2.1.3 國內外情緒分析研究現況

在國際層面上，情緒分析已經成為人機互動（Human-Computer Interaction, HCI）領域的熱門研究方向。許多知名的大學和研究機構都投入了大量資源進行相關研究。例如，MIT 媒體實驗室開發了多種情緒識別技術，包括透過心率變異性（Heart Rate Variability）來推斷情緒狀態的方法，以及利用機器學習分析臉部表情的系統。在產業應用方面，Amazon、Microsoft、Google 等科技巨頭也纷纷推出了基於情緒分析的產品和服務，如 Amazon Alexa 的情感識別功能、Microsoft Azure 的情緒 API 以及 Google Cloud 的情緒分析工具等。

在國內方面，台灣的大學和研究機構在情緒分析領域也有不少研究成果。例如，國立臺灣大學的研究團隊發表了多篇關於中文情緒分析的論文，提出了一系列適合中文文字特性的情感分類方法。此外，國立中山大學、成功大學等學校也都有相關的研究計畫，專注於開發適合中文語言環境的情緒識別系統。這些研究為本研究提供了重要的技術參考和理論基礎。

## 2.2 臉部表情辨識技術

### 2.2.1 Ekman 基本情緒理論

如前所述，Ekman 和 Friesen（1971）提出的六種基本情緒理論是臉部表情辨識研究的基石。他們的研究表明，這六種基本情緒在不同的文化和種族之間具有很高的一致性，人類能夠準確地從臉部表情中識別這些情緒。這一發現為後來的自動化臉部表情辨識系統提供了重要的理論依據。

Ekman 還開發了臉部動作編碼系統（FACS），該系統將臉部表情分解為多個動作單元（Action Units, AUs），並透過這些動作單元的組合來描述各種各樣的臉部表情。例如，「眉毛上揚」對應於 AU1+2，「嘴角上揚」對應於 AU12 等。FACS 系統後來被廣泛應用於臉部表情辨識系統的設計中，許多研究者和開發者都會參考 FACS 來定義表情特徵。

### 2.2.2 傳統機器學習方法

在深度學習技術普及之前，傳統的機器學習方法是臉部表情辨識的主流方法。這些方法通常包含以下幾個步驟：臉部偵測、特徵提取以及分類。

在臉部偵測方面，Viola 和 Jones（2001）提出的 Viola-Jones 演算法是最具代表性的方法。該演算法利用 Haar-like 特徵和級聯分類器，能夠在即時環境中快速且準確地偵測人臉。雖然 Viola-Jones 演算法在準確率上不及後來的深度學習方法，但其高效率使其在當時被廣泛應用。

在特徵提取方面，研究者們提出了多種手動設計的特徵表示方法，包括幾何特徵、外觀特徵以及動態特徵等。幾何特徵（Geometric Features）透過描述臉部關鍵點（如眼睛、嘴巴、鼻子等）的位置和相互關係來表示表情資訊。外觀特徵（Appearance Features）則是透過分析臉部圖像的整體外觀變化來捕捉表情資訊，常用的方法包括 Gabor 小波、主成分分析（PCA）以及局部二值模式（LBP）等。動態特徵（Dynamic Features）則是考慮了表情變化的時間維度，適合用於分析連續的臉部表情序列。

在分類階段，常用的機器學習演算法包括支援向量機（Support Vector Machine, SVM）、隨機森林（Random Forest）、類神經網路（Neural Network）以及隱馬爾可夫模型（Hidden Markov Model, HMM）等。這些方法在當時的標準數據集上（如 JAFFE、CK+）取得了不錯的效果，但往往受限於特徵設計的複雜度和泛化能力。

### 2.2.3 深度學習 CNN 方法

近年來，卷積神經網路（Convolutional Neural Network, CNN）的興起徹底改變了臉部表情辨識領域的研究格局。與傳統的機器學習方法相比，CNN 能夠自動從原始圖像中學習有用的特徵表示，無需人工設計特徵，這大大提高了系統的靈活性和泛化能力。

目前最受歡迎的臉部表情辨識 CNN 架構包括 VGGNet、ResNet、MobileNet 等。這些網路架構通常會在預訓練的基礎上進行微調（Fine-tuning），以適應表情分類的任務。此外，為了解決表情數據集規模有限的問題，研究者們也發展出了多種資料增強技術，包括隨機翻轉、旋轉、縮放、色彩變換等，這些技術能夠有效地提高模型的魯棒性。

在應用層面，許多研究者也嘗試將 CNN 與其他技術相結合，以進一步提升表情辨識的準確率和效率。例如，將 CNN 與長短期記憶網路（Long Short-Term Memory, LSTM）相結合，可以同時捕捉表情的空間特徵和時間特徵，適合處理連續的影片序列。

### 2.2.4 即時臉部偵測技術

隨著 Web 應用程式的普及，開發基於瀏覽器的即時臉部偵測系統成為了一個重要的研究方向。Google 開發的 MediaPipe 是一個功能強大的多媒體處理框架，提供了高效的人臉偵測和臉部特徵點追蹤功能。MediaPipe Face Mesh 能夠在瀏覽器中即時偵測 468 個臉部關鍵點，且延遲極低，非常適合用於需要即時反饋的應用場景。

另一個受歡迎的選擇是 face-api.js，這是一個基於 TensorFlow.js 的臉部辨識 JavaScript 庫，提供了多種功能包括臉部偵測、臉部比對、表情分類等。face-api.js 的優點是易於整合到現有的 Web 專案中，且支援多種預訓練模型。

Apple 的 Vision Framework 同樣提供了強大的臉部偵測能力，包括臉部特徵點偵測、表情分類以及臉部追蹤等功能。雖然 Vision Framework 主要面向 iOS 和 macOS 應用，但其高效率和高準確率也使其成為行動應用開發的良好選擇。

本研究在表情分析模組的設計中，選擇採用 MediaPipe Face Mesh 作為主要的臉部偵測工具。主要考量包括：其一，MediaPipe 提供了開源且高效的解決方案；其二，其 468 個臉部關鍵點能夠提供足夠的精細度來分析表情變化；其三，MediaPipe 支援多種平台，包括 Web、Android 以及 iOS，有利於未來的擴展。

## 2.3 語音情緒分析

### 2.3.1 語音特徵提取

語音情緒分析是透過分析語音信號中的各種 acoustic features 來推斷說話者的情緒狀態。多年來，研究者們已經提出了大量的語音特徵，這些特徵可以大致分為以下幾類：

**頻率特徵（Frequency Features）：** 這類特徵主要描述語音信號的頻率特性，常用的包括：
- 基頻（Fundamental Frequency, F0）：反映聲帶振動的基本頻率，與聲音的音高密切相關
- 語音共振峰（Formants）：反映口腔和聲道的共鳴特性
- 頻譜中心（Spectral Centroid）：頻譜的重心位置，反映聲音的亮度
- 頻譜滾降點（Spectral Rolloff）：頻譜能量下降到特定比例的位置

**能量特徵（Energy Features）：** 這類特徵描述語音信號的能量變化，常用的包括：
- 短時能量（Short-term Energy）：反映聲音的音量大小
- 能量變異數（Energy Variance）：反映能量的波動程度
- 過零率（Zero Crossing Rate）：信號通過零點的頻率，與語音的清濁音特性相關

**時間特徵（Temporal Features）：** 這類特徵描述語音信號的時間結構，常用的包括：
- 語速（Speech Rate）：單位時間內說出的音節數
- 停頓比例（Pause Ratio）：停頓時間佔總時間的比例
- 持續時間（Duration）：音節或詞語的持續時間

**MFCC（Mel-Frequency Cepstral Coefficients）：** MFCC 是語音識別領域最廣泛使用的特徵之一，它模擬了人類聽覺系統對頻率的感知特性。MFCC 能夠有效地捕捉語音的頻譜包絡資訊，且對於雜訊具有較強的魯棒性。在情緒分析中，MFCC 同樣被廣泛使用。

### 2.3.2 語音情緒資料庫

為了促進語音情緒分析的研究，許多研究者已經建立了多種語音情緒資料庫。這些資料庫通常包含帶有情緒標註的語音樣本，是訓練和評估情緒分類模型的重要資源。

**EMODB（Berlin Database of Emotional Speech）：** 這是最常用的語音情緒資料庫之一，由柏林工業大學建立。EMODB 包含 10 位演員錄製的 800 個德語語句，分別標註為七種情緒：憤怒、無聊、厭惡、恐懼、快樂、中性以及悲傷。

**IEMOCAP（Interactive Emotional Motion Capture Database）：** 這是一個規模更大的多模態資料庫，由南加州大學建立。IEMOCAP 包含 10 位演員錄製的約 12 小時的語音和視訊資料，涵蓋多種情緒類型。

**RAVDESS（Ryerson Audio-Visual Database of Emotional Speech and Song）：** 這個資料庫包含 24 位演員錄製的語音和歌曲樣本，涵蓋多種情緒和不同的表達方式。

在中文語音情緒資料庫方面，近年來也有多個中文資料庫被建立，為中文語音情緒分析的研究提供了重要的資源。

### 2.3.3 語音情緒分類模型

傳統的語音情緒分類方法通常基於上述的聲學特徵，配合 SVM、決策樹或隨機森林等機器學習分類器進行情緒識別。近年來，深度學習方法，特別是 CNN 和循環神經網路（RNN），在語音情緒分類任務中也取得了很好的效果。

CNN 能夠自動學習語音頻譜圖中的局部模式，而 RNN，特別是 LSTM，則能夠捕捉語音信號的時間動態特性。研究者們也嘗試將 CNN 和 LSTM 相結合，形成了 CNN-LSTM 或 LSTM-CNN 的混合架構，以同時利用空間和時間特徵。

近年來，基於注意力機制（Attention Mechanism）的Transformer架構也開始被應用於語音情緒分析領域，並展現出優異的性能。

## 2.4 文字情感分析

### 2.4.1 NLP 情感分析技術

文字情感分析（Sentiment Analysis）是自然語言處理（NLP）領域的一個重要分支，旨在自動識別和提取文字中表達的情感傾向。情感分析可以分為多個層次，包括文件層級（Document-level）、語句層級（Sentence-level）以及 Aspect 層級的情感分析。

文件層級的情感分析旨在判斷整篇文檔的情感傾向，是正面、負面還是中性。這是最基礎的情感分析任務，廣泛應用於產品評論分析、輿情監測等場景。

語句層級的情感分析則聚焦於分析單一語句的情感傾向。與文件層級相比，語句層級的分析更能捕捉細緻的情感變化，適合用於對話情感分析。

Aspect 層級的情感分析則進一步識別文字中具體評價的面向（Aspect）和其情感傾向。例如，在評論「這家餐廳的服務很好，但食物普通」中，Aspect 層級的分析會識別出「服務」為正面，「食物」為中性。

### 2.4.2 情感詞典與機器學習

在情感分析的早期研究中，情感詞典（Sentiment Lexicon）方法是主流。情感詞典是一個包含大量詞語及其情感極性和強度的詞典，如英文的 SentiWordNet、AFINN、LIWC，以及中文的 NTUSD 等。利用情感詞典進行情感分析的方法相對簡單：先將文字分詞，然後查詢每個詞的情感值，最後透過彙總計算得到整體的情感傾向。

機器學習方法的興起為情感分析帶來了新的可能性。傳統的機器學習方法通常需要手動設計特徵，包括詞袋模型（Bag of Words）、TF-IDF、N-gram 等文字表示方法，配合 SVM、朴素貝葉斯（Naive Bayes）或邏輯回歸（Logistic Regression）等分類器進行情感分類。

近年來，深度學習方法，特別是各類預訓練語言模型，已經成為情感分析的主流方法。BERT、RoBERTa、ERNIE 等預訓練模型能夠學習豐富的語言表示，在下游的情感分類任務上展現出優異的性能。

### 2.4.3 對話情感分析應用

對話情感分析是情感分析的一個重要應用場景，專注於分析對話過程中的情感變化和動態發展。與單純的文本情感分析相比，對話情感分析需要考慮更多的因素，包括對話歷史、發話者關係、對話情境等。

在社交機器人和客服系統中，對話情感分析有著廣泛的應用。例如，一個智能客服系統可以透過分析使用者的文字輸入來判斷其情緒狀態，並據此調整回覆的策略。當檢測到使用者情緒負面時，系統可以採取更加安撫性的回覆方式，或者及時轉接人工客服。

在心理健康領域，對話情感分析也被用於監測使用者的情緒狀態，特別是用於早期識別憂鬱症、焦慮症等情緒障礙。透過分析使用者的日常對話文字，相關系統可以發現情緒異常的跡象，並及時提醒或建議尋求專業協助。

## 2.5 多模態情緒識別

### 2.5.1 多模態融合方法

多模態情緒識別是指結合兩種或多種不同類型的輸入資料（如表情、語音、文字等）來進行情緒識別的方法。與單一模態相比，多模態方法能夠提供更加豐富和全面的資訊，從而提高情緒識別的準確率和魯棒性。

多模態融合方法可以分為早期融合（Early Fusion）、晚期融合（Late Fusion）以及混合融合（Hybrid Fusion）三種：

**早期融合（Early Fusion）：** 又稱特徵級融合（Feature-level Fusion），是指在特徵提取階段就將不同模態的特徵向量連接成一個統一的特徵向量，然後輸入到單一的分類器中進行處理。早期融合的優點是能夠捕捉不同模態之間的交互關係，但其缺點是需要處理不同模態資料的時間對齊問題。

**晚期融合（Late Fusion）：** 又稱決策級融合（Decision-level Fusion），是指對每個模態分別進行獨立的處理和分類，獲得各模態的分類結果或機率分佈，然後透過加權平均、投票或更複雜的融合策略來獲得最終的結果。晚期融合的優點是各模組可以獨立設計和優化，且對不同模態資料的時間同步要求較低。

**混合融合（Hybrid Fusion）：** 結合了早期融合和晚期融合的優點，通常採用多層次的融合架構，在不同層次上進行資訊的整合。

近年來，深度學習技術也為多模態融合帶來了新的可能性。注意力機制（Attention Mechanism）特別適合用於多模態學習，它能夠動態地調整不同模態在融合過程中的權重，使模型能夠根據輸入資料的特點自動選擇最相關的資訊。

### 2.5.2 現有多模態情緒識別系統分析

近年來，已有多個多模態情緒識別系統被開發出來，這些系統在不同的應用場景中展現出了各自的優勢。

在學術研究方面，許多研究者提出了各種多模態情緒識別框架。例如，Zadeh 等人（2017）提出的 Multi-view Sequenced Memory Network（MS-MVN）能夠同時處理表情、語音和文字三種模態的時序資料，並透過記憶網路來捕捉長期的依賴關係。Poria 等人（2017）則提出了一種基於深度信念網路（Deep Belief Network）的多模態融合方法，在多個標準數據集上取得了領先的性能。

在開源工具方面，許多公司和研究機構也提供了多模態情緒識別的解決方案。例如，Google 的 Cloud Natural Language API 提供了文字情感分析功能，Microsoft Azure 的 Face API 提供了表情識別功能，而 Amazon Polly 雖然主要是文字轉語音服務，但也提供了情感表達的功能。這些工具雖然功能強大，但往往是收費的且不支援完全的自定義。

在情感計算領域，還有一些專門針對情緒識別優化的框架和庫，如 DEAP（Dataset for Emotion Analysis using Physiological signals）等，這些資源為研究者提供了重要的技術支持。

### 2.5.3 本研究與現有系統之差異

與現有的多模態情緒識別系統相比，本研究具有以下幾個特點：

首先，本研究專注於開發一個完整的 Web 應用程式，而非僅僅是演算法或工具。這意味著我們的系統不僅包含了情緒分析的核心技術，還包含了完整的用戶介面、通訊功能以及數據管理功能，能夠提供一站式的解決方案。

其次，本研究在情緒分類的設計上採用了更為簡潔的 High/Normal/Low 三級分類方式。雖然這種分類方式犧牲了情緒類型的多樣性，但卻能夠提供更加明確和實用的輸出結果。這種設計特別適合非專業使用者，能夠讓他們快速理解自己的情緒狀態，而無需理解複雜的情緒類型。

第三，本研究在系統設計上特別強調即時性和易用性。透過採用 TensorFlow.js 等技術，我們能夠在瀏覽器中直接進行情緒分析，無需將資料上傳到伺服器處理。這不僅能夠降低延遲，還能夠保護使用者的隱私。

最後，本研究的一個重要目標是實現個人情緒構成分析功能。透過收集並分析使用者在多次使用中的情緒數據，我們希望能夠建立個人情緒檔案，幫助使用者了解自己的情緒模式和變化趨勢。這種功能在現有的系統中較少見到，卻對於個人情緒健康管理具有重要的價值。

## 2.6 相關產品分析

### 2.6.1 現有視訊配對產品

在市場上，已有多款視訊配對產品，它們各自針對不同的使用場景和目標群體。

**Omegle：** 這是最早期的隨機視訊配對服務之一，使用者可以與陌生人進行隨機的視訊聊天。Omegle 的設計簡單直接，但缺乏任何形式的情緒分析或配對優化功能。

**Chatroulette：** 類似於 Omegle，也是提供隨機視訊配對服務。雖然在高峰時期曾擁有大量用戶，但由於缺乏有效的內容審核機制，近年來的使用率有所下降。

**Tandem：** 這是一款專注於語言學習的視訊配對應用，使用者可以透過與母語者視訊聊天來學習語言。雖然 Tandem 沒有情緒分析功能，但其在配對演算法上的設計（如根據學習興趣、配對時間等）值得參考。

**Hello:** 這是一款基於 AI 的視訊約會應用，宣稱使用 AI 來分析用戶的相容性。雖然具體的技術細節不得而知，但其「AI 配對」的概念與本研究有一定相似之處。

### 2.6.2 情緒分析相關應用

情緒分析技術已經被廣泛應用於各種產品和服務中。

**情緒追蹤應用：** 如 Moodpath、Daylio 等應用可以讓使用者每日記錄自己的情緒狀態，並生成情緒報告。這類應用主要依賴使用者的主觀輸入，缺少自動化的情緒識別功能。

**客服機器人：** 許多企業的客服系統已經開始使用情緒分析技術來識別客戶的情緒狀態，並據此調整服務策略。例如，當檢測到客戶情緒負面時，系統可能會優先轉接人工客服。

**健康監測：** 一些健康應用開始整合情緒分析功能，特別是與心理健康相關的應用。例如，一些憂鬱症篩檢應用會分析使用者的文字輸入來識別潛在的情緒問題。

**教育科技：** 在線學習平台開始使用情緒分析來監測學生的學習狀態。當檢測到學生感到困惑或挫敗時，系統可以提供額外的幫助或調整教學內容。

### 2.6.3 產品優缺點分析

综合上述產品的分析，我們可以總結出以下優點和缺點：

**現有產品的優點：**
- 功能明確，專注於特定的使用場景
- 用戶介面設計相對成熟
- 技術實現相對穩定

**現有產品的缺點：**
- 很少整合完整的情緒分析功能
- 缺乏個人情緒歷史的追蹤和分析
- 對用戶隱私的保護不夠重視
- 缺乏開源或可自定義的解決方案

基於這些分析，本研究的 MeetLove 系統在設計上特別注重以下幾點：完整的情緒分析功能、重視用戶隱私、提供個人情緒歷史追蹤、以及提供開源的可自定義解決方案。我們希望透過這些特點，能夠為用戶提供更好的使用體驗和價值。
