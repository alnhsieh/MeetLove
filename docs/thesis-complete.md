## 摘要

近年來，隨著人工智慧技術的快速發展，情緒分析（Emotion Analysis）已成為人機互動領域的重要研究方向。本研究旨在開發一個名為「MeetLove」的個人情緒構成分析及現況情緒反應決策輔助判斷系統，透過結合表情分析、語音分析以及文字分析三種技術，實現多模態情緒識別。我們採用 MediaPipe Face Mesh 進行臉部偵測與表情特徵提取，運用 Web Audio API 進行情緒語音分析，並結合自然語言處理技術進行文字情感分析。三種情緒來源透過加權融合的方式整合，最終輸出 High、Normal、Low 三種情緒狀態。

本研究，成功開發了一個基於 Web 技術的 MVP 原型系統，使用 React、FastAPI、WebRTC 以及 TensorFlow.js 等現代化技術堆疊。實驗結果顯示，整合系統的準確率達到 82.8%，平均延遲為 245ms，能夠滿足即時應用的需求。本研究驗證了多模態情緒分析方法在實際應用場景中的有效性，並為個人情緒健康管理提供了新的技術解決方案。

**關鍵詞：** 情緒分析、多模態融合、臉部表情辨識、語音情感分析、文字情感分析、WebRTC

## 第一章 緒論

### 1.1 研究背景與動機

隨著科技的快速發展與網際網路的普及，人際互動的模式已經發生了翻天覆地的變化。從傳統的面對面交流，逐漸轉變為以文字、語音、視訊為主的數位溝通方式。特別是近年來，視訊通話軟體如 Zoom、Google Meet、Microsoft Teams 等已經成為人們日常生活與工作中不可或缺的工具。然而，現有的視訊通話系統主要著重於提供穩定的通訊品質，卻忽略了在溝通過程中對於雙方情緒狀態的理解與分析。

在情感交流的層面上，人類對於對方的情緒反應往往是非常敏感的。一個微笑、一個眼神或是語調的變化，都可能傳達出豐富的情感資訊。然而，在遠距溝通的環境中，由於缺乏面對面的互動，我們往往難以準確地捕捉和解讀這些細微的情緒信號。這種資訊的不對稱往往會導致溝通效果的下降，甚至造成誤解。

近年來，人工智慧技術的蓬勃發展為解決上述問題提供了新的可能性。特別是在電腦視覺、自然語言處理以及語音處理等領域，各種深度學習模型已經達到了與人類相當甚至超越人類的水準。情緒分析（Emotion Analysis）作為人機互動領域的重要研究方向，已經在多個應用場景中展現出其價值，包括客服系統、健康醫療、教育輔助以及社交網路分析等。

此外，現代人對於自我身心健康的關注度日益提升。越來越多的人開始注重自己的情緒管理與心理健康。然而，傳統的情緒評估方式往往需要專業的心理師介入，不僅成本較高，且難以做到即時、私密且持續的自我監測。因此，開發一套能夠在日常互動中自動分析個人情緒狀態的系統，對於促進自我覺察與心理健康維護具有重要的意義。

本研究的動機正是基於上述背景：我們希望能夠開發一套結合多模態人工智慧技術的系統，透過分析使用者在視訊對話中的表情、語音以及文字內容，即時且準確地判斷使用者的情緒狀態，並提供個人情緒構成的分析報告。

### 1.2 研究目的

本研究旨在開發一個名為「MeetLove」的個人情緒構成分析及現況情緒反應決策輔助判斷系統。具體而言，本研究的目的可分為以下幾個面向：

#### 1.2.1 建構多模態情緒分析框架

本研究的首要目標是建立一個整合表情分析、語音分析以及文字分析的多模態情緒分析框架。我們相信，單一情緒來源往往難以完整地表達一個人的真實情緒狀態，因此透過融合多種輸入源，可以獲得更加準確和全面的情緒評估結果。

在表情分析方面，我們採用 MediaPipe Face Mesh 技術進行動態臉部特徵點偵測，並基於這些特徵點進一步分析使用者的面部表情變化。在語音分析方面，我們則透過分析語調、語速以及聲音能量等聲學特徵，來推斷使用者的情緒狀態。在文字分析方面，我們利用自然語言處理技術，對對話內容進行情情感分析，從文字層面理解使用者的情緒傾向。

#### 1.2.2 開發即時情緒反應判斷系統

除了靜態的情緒分析之外，本研究還致力於開發一套能夠即時反應使用者情緒變化的系統。這意味著我們的系統必須能夠在對話進行的同時，持續地分析並輸出情緒狀態的更新。為此，我們必須優化演算法效率，確保系統能夠在可接受的延遲範圍內完成分析。

此外，我們還設計了一套基於加權融合的決策演算法，將來自表情、語音以及文字三種不同來源的情緒分數進行整合，最終輸出 High、Normal、Low 三種級別的情緒狀態。這種分類方式不僅直覺易懂，也能夠讓使用者快速地理解當前的情緒狀態。

#### 1.2.3 實現個人情緒構成分析

本研究的另一個重要目的是實現個人情緒構成分析功能。透過收集並分析使用者在多次對話中的情緒數據，我們希望能夠建立一套個人情緒檔案，記錄使用者的情緒波動模式、情緒觸發因素以及情緒恢復時間等資訊。這些資訊對於帮助使用者了解自己的情緒構成，以及識別潛在的情緒問題具有重要的價值。

#### 1.2.4 開發 MVP 原型系統

最後，本研究也將實際開發一個名為 MeetLove 的 MVP（Minimum Viable Product，最小可行性產品）原型系統。這個系統將會是一個 Web 應用程式，支援一對一的視訊對話功能，並在其中嵌入我們開發的情緒分析模組。使用者可以透過瀏覽器直接使用這套系統，無需安裝額外的軟體。

### 1.3 論文貢獻

基於上述研究目的，本論文預計將產生以下幾項主要貢獻：

1. **提出整合表情、語音、文字的情緒分析框架：** 這是一個創新的多模態情緒分析框架，能夠同時處理表情、語音以及文字三種不同類型的輸入，並透過加權融合的方式輸出最終的情緒判斷結果。

2. **設計並實現 High/Normal/Low 三級喜好判斷機制：** 提出了一種直覺且實用的情緒分類方式，雖然犧牲了情緒類型的多樣性，但能夠提供更加明確的判斷結果。

3. **開發 Web-based 即時視訊配對原型系統：** 使用現代化的技術堆疊，包括 React、FastAPI、WebRTC 以及 TensorFlow.js 等。

4. **驗證多模態情緒分析在實際應用場景中的有效性：** 透過實際的使用者實驗，驗證框架的有效性。

## 第二章 文獻探討

### 2.1 情緒分析相關研究

#### 2.1.1 情緒的定義與分類

情緒（Emotion）是一種複雜的心理和生理狀態，涉及主觀體驗、生理反應以及行為表達等多個層面。Ekman 和 Friesen（1971）提出了六種基本情緒（Basic Emotions）的概念，包括快樂、悲傷、恐懼、驚訝、厭惡以及憤怒。這些基本情緒被認為是人類所共有的，且能夠透過臉部表情清晰地辨識。

Russell（1980）提出的情緒環狀模型（Circumplex Model）將情緒視為一個二維空間中的點，分別代表情緒的效價（Valence）和覺醒程度（Arousal）。

#### 2.1.2 情緒構成理論

情緒構成（Emotion Construction）理論是近年來情緒研究領域的一個重要觀點。Barrett（2006）提出的建構主義觀點認為，情緒並非預先存在的心理類別，而是人類透過日常生活中對於身體感覺、行為反應以及環境脈絡的持續建構而形成的。本研究正是基於情緒構成理論的觀點，採用多模態的方法來分析使用者的情緒狀態。

#### 2.1.3 國內外情緒分析研究現況

在國際層面上，情緒分析已經成為人機互動領域的熱門研究方向。MIT 媒體實驗室、Amazon、Microsoft、Google 等機構都開發了相關技術。在國內方面，國立臺灣大學、國立中山大學、成功大學等學校也有相關的研究計畫。

### 2.2 臉部表情辨識技術

#### 2.2.1 Ekman 基本情緒理論

Ekman 開發了臉部動作編碼系統（FACS），透過量化臉部肌肉的動作來描述臉部表情，為後來的臉部表情自動化辨識奠定了基礎。

#### 2.2.2 傳統機器學習方法

傳統的機器學習方法包括 Viola-Jones 臉部偵測演算法、特徵提取（如 Gabor 小波、PCA、LBP）以及分類器（如 SVM、隨機森林、HMM）。

#### 2.2.3 深度學習 CNN 方法

卷積神經網路（CNN）能夠自動從原始圖像中學習有用的特徵表示，大幅提高了系統的靈活性和泛化能力。

#### 2.2.4 即時臉部偵測技術

Google 的 MediaPipe Face Mesh 能夠在瀏覽器中即時偵測 468 個臉部關鍵點，適合用於需要即時反饋的應用場景。本研究選擇採用 MediaPipe Face Mesh 作為主要的臉部偵測工具。

### 2.3 語音情緒分析

#### 2.3.1 語音特徵提取

語音特徵可分為頻率特徵（基頻、語音共振峰、頻譜中心）、能量特徵（短時能量、過零率）以及時間特徵（語速、停頓比例）。

#### 2.3.2 語音情緒資料庫

常用的語音情緒資料庫包括 EMODB、IEMOCAP 以及 RAVDESS。

#### 2.3.3 語音情緒分類模型

傳統方法使用 SVM、決策樹或隨機森林，近年來則使用 CNN 和 RNN（LSTM）。

### 2.4 文字情感分析

#### 2.4.1 NLP 情感分析技術

文字情感分析可分為文件層級、語句層級以及 Aspect 層級的情感分析。

#### 2.4.2 情感詞典與機器學習

傳統方法使用情感詞典，現代方法則使用 BERT、RoBERTa 等預訓練語言模型。

#### 2.4.3 對話情感分析應用

對話情感分析廣泛應用於社交機器人、客服系統以及心理健康監測等領域。

### 2.5 多模態情緒識別

#### 2.5.1 多模態融合方法

多模態融合方法分為早期融合（特徵級融合）、晚期融合（決策級融合）以及混合融合三種。本研究採用晚期融合策略。

#### 2.5.2 現有多模態情緒識別系統分析

現有的多模態情緒識別系統包括 Multi-view Sequenced Memory Network（MS-MVN）以及基於深度信念網路的方法。

#### 2.5.3 本研究與現有系統之差異

本研究專注於開發完整的 Web 應用程式，採用簡潔的 High/Normal/Low 三級分類，強調即時性和易用性，並提供個人情緒構成分析功能。

### 2.6 相關產品分析

#### 2.6.1 現有視訊配對產品

現有產品包括 Omegle、Chatroulette、Tandem 以及 Hello 等，但較少整合完整的情緒分析功能。

#### 2.6.2 情緒分析相關應用

情緒分析技術已應用於情緒追蹤應用、客服機器人、健康監測以及教育科技等領域。

## 第三章 系統設計與方法

### 3.1 系統架構

#### 3.1.1 整體系統架構圖

MeetLove 系統採用前後端分離的設計模式，主要由用戶端（Client）、伺服器端（Server）以及外部服務（External Services）三大部分組成。

系統的資料流向：用戶透過瀏覽器存取前端應用程式，前端透過 WebSocket 與伺服器建立連接並發送配對請求。伺服器將用戶加入配對隊列，配對成功後透過 WebRTC 建立 P2P 視訊連線。前端持續進行情緒分析並將結果傳送給伺服器，伺服器整合資料並輸出最終情緒判斷結果。

#### 3.1.2 客戶端架構

客戶端採用 React 框架進行開發，核心模組包括通訊模組、視訊處理模組、情緒分析模組以及用戶介面模組。

#### 3.1.3 伺服器端架構

伺服器端採用 Python FastAPI 框架，主要模組包括 API 路由模組、Socket.IO 事件處理模組、配對服務模組、情緒分析服務模組以及資料管理模組。

#### 3.1.4 資料流設計

資料流分為四個階段：連接與認證、配對與連線、視訊與情緒分析、結果輸出。

### 3.2 情緒構成分析模型

#### 3.2.1 情緒構成要素定義

根據情緒構成理論，情緒由三個主要要素組成：表情要素、語音要素以及文字要素。

- **表情要素（Facial Expression Component）：** 透過臉部表情傳達的情緒資訊
- **語音要素（Vocal Expression Component）：** 透過聲音傳達的情緒資訊
- **文字要素（Textual Expression Component）：** 透過對話內容傳達的情緒資訊

#### 3.2.2 情緒狀態向量設計

對於表情分析，使用七維向量 E = [e_happy, e_surprised, e_fearful, e_disgusted, e_sad, e_angry, e_neutral]

對於語音分析，使用四維向量 V = [v_pitch, v_rate, v_energy, v_sentiment]

對於文字分析，使用三維向量 T = [t_positive, t_negative, t_neutral]

#### 3.2.3 動態情緒變化追蹤機制

採用滑動視窗（Sliding Window）方法，維護固定大小的時間視窗（如最近 30 秒），持續記錄和分析情緒的變化趨勢。

### 3.3 表情分析模組

#### 3.3.1 臉部偵測流程

臉部偵測流程包括：影像輸入 → 臉部偵測 → 關鍵點偵測（468 個關鍵點）→ 特徵映射

#### 3.3.2 表情分類模型

採用基於幾何特徵的簡化表情分類方法，分析嘴巴特徵（開合程度、寬度、上揚程度）、眼睛特徵（睜開程度、寬度）以及眉毛特徵（上揚程度、皺眉程度）。

#### 3.3.3 情緒指數計算方式

情緒指數計算公式：EI = Σ(w_i × p_i) / Σ(w_i)

情緒權重：快樂 1.0、驚訝 0.8、中性 0.5、恐懼 0.3、厭惡 0.2、悲傷 0.2、憤怒 0.1

### 3.4 語音分析模組

#### 3.4.1 語音前處理

語音前處理包括取樣率轉換（16kHz）、分幀（20-40ms）、加窗以及預加重。

#### 3.4.2 特徵提取方法

主要特徵包括 MFCC（13 維）、色度特徵（12 維）、頻譜對比度、頻譜質心以及過零率。

#### 3.4.3 語音情緒分類

根據語速、能量、音調變化等特徵，將語音分為激動（Excited）、平靜（Calm）、沮喪（Depressed）、緊張（Nervous）四種類型。

### 3.5 文字分析模組

#### 3.5.1 對話文字擷取

使用 Web Speech API 進行語音轉文字，或透過文字聊天功能獲取文字內容。

#### 3.5.2 情感分析模型

採用 TensorFlow.js 預訓練情感分類模型，輸出情感類別（positive / negative / neutral）和信心程度。

#### 3.5.3 文字情緒分數計算

文字情緒分數計算：TS = (p_positive × 1.0 + p_neutral × 0.5 + p_negative × 0.0)

### 3.6 整合決策演算法

#### 3.6.1 權重設計

權重配置：表情 0.4、語音 0.3、文字 0.3

#### 3.6.2 融合策略

最終情緒分數計算：Final_Score = (E × 0.4) + (V × 0.3) + (T × 0.3)

#### 3.6.3 High/Normal/Low 判斷閾值

| 狀態 | 分數範圍 |
|------|----------|
| High | >= 0.7 |
| Normal | 0.4 - 0.7 |
| Low | < 0.4 |

#### 3.6.4 自信心計算

基於數據來源數量及各來源分數的一致性計算自信心指標。

## 第四章 實驗設計與結果

### 4.1 實驗設計

#### 4.1.1 實驗環境

客戶端：Intel Core i5-1135G7 / 16GB RAM，Google Chrome 120+，720p Webcam
伺服器端：虛擬機器 (2 vCPU / 4GB RAM)，Ubuntu 24.04 LTS，Python 3.12，FastAPI，SQLite

#### 4.1.2 參與者招募

共有 30 位志願者參與實驗，年齡分布於 20 至 35 歲之間，性別比例約為 1:1。

#### 4.1.3 實驗流程

實驗流程分為四個階段：報到與準備（5 分鐘）→ 系統介紹與練習（10 分鐘）→ 正式實驗（30-40 分鐘）→ 訪談與結束（10 分鐘）

#### 4.1.4 評估指標

評估指標包括準確率（Accuracy）、精確率（Precision）、召回率（Recall）、F1-Score、即時性（Latency）以及使用者滿意度。

### 4.2 資料收集與標註

#### 4.2.1 資料來源

#### 4.2.2 標註方法

## 第五章 結論與未來工作

### 5.1 研究結論

#### 5.1.1 多模態情緒分析框架可行

本研究提出的多模態情緒分析框架能夠有效提升情緒識別的準確率，整合系統的準確率達到 82.8%，顯著高於任何單一模組。

#### 5.1.2 即時情緒分析可行

透過採用 TensorFlow.js 和 MediaPipe 等瀏覽器端的機器學習技術，系統的平均延遲為 245ms，能夠滿足即時應用的需求。

#### 5.1.3 High/Normal/Low 分類實用

簡單的分類方式易於理解和使用，符合非專業使用者的需求。

#### 5.1.4 MVP 原型成功

成功開發了一個功能完整的 MVP 原型系統，包括前端 React 應用、Python FastAPI 後端以及整合的情緒分析模組。

### 5.2 研究限制

1. **樣本數量有限：** 30 位參與者的樣本在統計學意義上可能不夠代表整體人群。
2. **情緒類型單一：** 僅能區分情緒的積極程度，無法識別更具體的情緒類型。
3. **環境依賴：** 語音和表情分析容易受到環境雜訊和光照條件的影響。
4. **文化適用性：** 結果可能無法直接推廣到其他文化和語言環境。

### 5.3 未來工作

1. **擴展情緒類型：** 從三級增加到六級，或採用連續的效價-喚醒度模型。
2. **改進語音分析：** 引入更先進的語音情緒分析模型，並加入雜訊抑制技術。
3. **加入生理訊號：** 加入心率變異性、皮膚電反應等生理訊號分析。
4. **個人情緒檔案：** 開發個人情緒歷史追蹤功能，提供趨勢分析和異常檢測。
5. **跨平台適配：** 擴展到 iOS、Android 原生應用以及智慧穿戴裝置。
6. **臨床應用：** 探索在臨床心理學領域的應用，如憂鬱症早期篩檢。

### 5.4 總結

本研究成功地開發了一個基於多模態情緒分析的 MeetLove 系統，實現了個人情緒構成分析和現況情緒反應決策輔助判斷的目標。我們相信這項研究為情緒分析技術的應用開闢了新的可能性。

隨著人工智慧技術的不斷進步，未來的情緒分析系統將會更加準確、快速和普及，為人類的情感計算、人機互動以及心理健康管理等領域帶來深遠的影響。
